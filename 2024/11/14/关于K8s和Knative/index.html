关于K8s和Knative

K8s

  • 组件架构：

    • 控制节点（master）

      四个重要组件：

      • ApiServer

        资源操作入口

      • Scheduler

        集群资源调度，按策略将Pod调度到node

      • ControllerManager

        维护集群状态

      • Etcd

        相当于K8s的数据库

    • 工作节点（node）

      提供环境：

      • Kubelet
      • KubeProxy
      • Docker

  • K8s控制器

    • Replicaset

      副本控制器（rs），控制由其管理的pod，使pod副本的数量始终维持在预设的个数；

      • 组成部分：

        • 用户期望的pod副本数

        • 标签选择器
        • pod资源模板

​ 

- **Deployment**

Kubectl

自带客户端，可用来直接操作K8s集群

  • K8s命令行工具，可以对其集群运行命令，部署应用，监测管理集群资源和查看日志；

    • 容器编排层
      1. 保证在多个主机上配置和管理容器，保障容器的运行；
      2. 自动分配保证负载的平衡；
      3. 根据需求自动增减容器的数量（弹性收缩）；
      4. 监控容器的健康状态，对出现问题的容器自动重启；
      5. 提供容器间的通信机制；
  • 语法

    使用以下语法从终端窗口运行 kubectl 命令：

    • command：指具体操作（create,get等）

    • 对类型相同的资源分组

一些区分

  • Kubectl

    • K8s命令行工具；

    • 操作K8s集群的工具，允许用户发送请求给K8s API Server管理集群中的资源；

  • Minikube

    • 本地集群工具，适于开发和测试；

    • 提供轻量级本地集群，简单的环境来运行容器化的应用；

  • Kubelet

    • 核心组件，节点上的代理程序；

    • 集群中每个节点的守护进程，直接和容器运行时交互；

  • Knative

    • 无服务平台
